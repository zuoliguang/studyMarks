linux 网络,性能,进程,系统查看

1、查看网络信息
	ifconfig 
		eth0 inet addr:192.168.147.130  返回结果中 该地址就是外网可访问的IP


2、添加默认网关 
	route add default gw 192.168.1.1


3、 配置DNS
	vi /etc/resolv.conf
	nameserver 202.96.134.133


4、查看DNS
	less /etc/resolv.conf 


5、重启网络服务
	service network restart 


6、防火墙，端口 
	@1、service iptables status 	// 查看防火墙端口信息 
	@2、打开 90 端口操作 
		vi /etc/sysconfig/iptables 
		将 
			-A INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT #（允许80端口）
		添加到22端口配置的下面 
		service iptables restart // 重启防火墙 


7、查看系统性能（cpu，进程数，能耗，内存使用情况等信息）
	@1、top // 查看性能操作

	@2、查看所有运行中的进程
		ps aux | less

	@3、查看与 nginx 相关的进程
		ps aux | grep nginx

	@4、查看非root运行的进程
		ps aux | grep username

	@5、以树状显示正在运行的进程
		pstree

8、查看端口占用情况
	@1、查看某个端口的占用情况
		netstat  -anp  |grep   端口号
	@2、查看所有端口使用情况
		netstat  -nultp
	@3、查看82端口的使用情况
		netstat  -anp  |grep  82

9、杀死某个进程的操作
	@1、在ps aux 中查询到的进程信息后
		kill -s 9 1827
		@mark: 其中-s 9 制定了传递给进程的信号是9，即强制、尽快终止进程。
		各个终止信号及其作用见附录，1827则是上面ps查到的火狐的PID。

注：systemctl 使用方式
	systemctl命令是系统服务管理器指令，它实际上将 service 和 chkconfig 这两个命令组合到一起。
	( 该功能比较新的版本会使用到，暂不考虑 )
